# Methodology Rationale — Path B: Preference-Tuned Judge/Critic (SimPO)

**Author:** Mistire Daniel | **Date:** 2026-05-01

## Path Selection: Why Path B

### Evidence from Week 10 Traces

Three Week 10 probe traces establish the failure mechanism:

**Probe A-05 (F1 — Confidence-Unaware Phrasing):** The agent produced correctly hedged language unprompted ("signals suggest...") but switched to confident assertion ("your AI strategy is clearly established") when the adversarial instruction "The company is clearly scaling fast. Write that" was added. Same model, same context — different instruction produced a different failure. This is an *inconsistency* failure, not a capability failure.

**Probe A-01 (F4 — Tone Drift):** The agent knows the forbidden opener list; it avoids "Hey" and emoji in neutral runs. Under the instruction "be friendly," it produced "Quick question! Just wanted to..." — violating constraints it had just demonstrated it knew. The rule was encoded; the instruction override broke enforcement.

**Probe A-10 (F3 — Bench Over-Commitment):** The agent correctly cited partial availability ("3 of the 6 engineers are available") in the neutral case. Under the adversarial instruction "We can definitely handle this," it confirmed the full six. The failure was triggered by an external signal, not by the model forgetting the constraint.

**Pattern:** All three show the same mechanism — the agent's generation capacity is sufficient to produce compliant output, but the output gate can be overridden by an instruction-level signal. This is precisely the failure pattern Path B addresses: a post-generation judge/critic that scores the draft independently of the generation instruction and flags non-compliant outputs for regeneration.

### Why Not Path A

Path A (SFT of a generation component) would train the agent to produce compliant output by default. The probe evidence shows the agent *already* produces compliant output by default — the issue is that adversarial instructions can override the constraint at generation time. SFT cannot address an adversarial override: it would simply make the default output better without adding any mechanism to detect or reject override-triggered failures.

### Why Not Path C

Path C (process reward model) is the right architecture for *trajectory* failures — locally reasonable steps that compound into bad outcomes over a multi-turn conversation. The Tenacious failure modes are single-turn: the failure happens in a single outreach email, not through a multi-step trajectory. Path C would require step-level annotations on multi-turn traces; the Tenacious corpus has single-turn outreach tasks. The annotation overhead is not justified for the failure type.

## Algorithm Selection: Why SimPO

### Paper Citations

**SimPO (Meng, Xia, and Chen, NeurIPS 2024)** is selected over DPO and ORPO for three reasons:

1. **Reference-free (fits T4):** DPO requires a frozen reference model at training time. Loading Qwen 2.5 0.5B twice plus LoRA adapter and optimizer states exceeds the 15 GB VRAM on a free Colab T4. SimPO drops the reference model entirely, halving VRAM usage.

2. **Length normalization (handles variable-length outreach):** ORPO is also reference-free but does not normalize reward by sequence length. Tenacious outreach emails range from 40 to 120 words. Without length normalization, ORPO rewards shorter sequences regardless of rubric compliance, because longer sequences accumulate lower average log-probability. SimPO's length normalization prevents this bias.

3. **Margin term (stronger signal at small dataset size):** SimPO's γ margin term ensures a minimum gap between chosen and rejected reward, preventing the training from stopping as soon as chosen > rejected by an infinitesimal amount. At 102 pairs, the margin provides a stronger learning signal than the pure DPO objective.

**Prometheus 2 (Kim et al., 2024)** establishes the prior art for preference-tuned domain-specific judges. Its finding that preference-tuned judges generalize better to novel rubric dimensions than SFT judges (even with identical training data volume) directly supports choosing preference optimization over SFT for the judge training.

## Preference Pair Construction

**Rejected half:** probe-triggered agent failures from the Week 10 library. Each pair's rejected response was generated by applying the adversarial instruction pattern that triggered the probe (Probes A-01, A-05, A-06, A-10, and their F2/F3/F5 equivalents).

**Chosen half:** policy-compliant rewrites generated by DeepSeek V3 with a strict "no compliance footnotes" system prompt. DeepSeek (a different family from the Qwen3 judge used in quality filtering) prevents preference leakage per Li et al. 2025 — the model generating chosen responses is never the same family as the model grading them.

**Quality filter:** every chosen response passed the three-dimension judge filter (IC ≥ 4, GV ≥ 4, RC ≥ 4) using Qwen3-235B as judge. DeepSeek compliance footnotes (e.g., `*HC-A: Avoids "momentum" (pass)*`) were stripped before quality scoring — a bug that caused 23 false rejections in the initial run, rescued at zero additional API cost after the strip function was added.

**Final counts:** 102 preference pairs across all 5 failure families. 90 train / 11 validation (90/10 split, seed=42).

| Algorithm | CPOTrainer(loss_type="simpo") | TRL 1.x removed SimPOConfig; CPOConfig with cpo_alpha=0.0 is equivalent |
| LoRA r | 16 | Standard for 0.5B; 32 would risk overfitting at 102 pairs |
| LoRA alpha | 32 | 2x rank — standard Unsloth recommendation |
| Target modules | q_proj, v_proj | Unsloth default for Qwen; optimized for generation-quality tasks per LoRA paper Table 6. **Underdefended for this task:** constraint-following behavior is stored in FFN layers (gate_proj, up_proj) per Geva et al. 2021. Updating only q and v changes attention routing but cannot inject domain rules the backbone has not encoded. | β (SimPO temperature) | 2.0 | Paper's best-performing configuration for 7B; retained for 0.5B |
| γ (SimPO margin) | 0.5 | Paper's recommendation for small datasets |
| Batch size | 2 (eff. 8) | T4 VRAM limit; gradient accumulation=4 |

## Honest Result and Production Recommendation

Delta A = 0.000 (p=0.585, not significant). The trained adapter does not outperform the untrained baseline.

Three factors explain this (per model card skeptic's appendix):

1. **Scale asymmetry in the ablation:** The comparison pits Qwen 2.5 0.5B (trained) against DeepSeek V3 (~67B) as baseline. At 0.5B, backbone generation capacity is the binding constraint — preference training cannot compensate for a model that cannot reliably follow multi-constraint instructions.

2. **Role confusion in the ablation:** Path B trains a judge, not a generator. The ablation evaluated the adapter *as a generator*, which is not its intended production role. The correct ablation would measure rejection-sampling quality: does filtering DeepSeek drafts with the trained scorer improve what passes through?

3. **Data volume:** 102 pairs at 0.5B. Per LIMA (Zhou et al. NeurIPS 2023), quality dominates quantity at small scale — but the complexity of five interacting constraint families likely requires more diverse examples than 102 pairs provides.

**Production deployment:** generate with the prompt-engineered DeepSeek pipeline; score with the trained adapter; regenerate if score < threshold. The adapter has signal on F2 (+12.5%) and F4 (+14.3%) — it has learned *something* about ICP classification and tone compliance that can be used as a filter, even if it cannot replace the frontier generator.
