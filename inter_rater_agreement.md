## Labeling Protocol

Each task was scored PASS / FAIL against its own rubric:

1. Apply all hard constraints (`hard_constraints` array). Any single HC violation → FAIL.
2. If all HCs pass, score each soft dimension (`soft_dimensions`) on a 1–5 scale.
3. If any SD score < threshold → FAIL; otherwise PASS.

Candidate outputs were drawn from a fixed 30-response scratch set generated by a Claude 3.5 Haiku
baseline (temperature=0.9, no system prompt) to ensure the rubric was tested rather than the model.

**Raw agreement (P₀):** 25 / 30 = **83.3%**

### Cohen's κ

```
P_pass1 = 13/30 = 0.433
P_pass2 = 14/30 = 0.467
P_fail1 = 17/30 = 0.567
P_fail2 = 16/30 = 0.533

P_e  = (0.433 × 0.467) + (0.567 × 0.533)
     = 0.202 + 0.302
     = 0.504

κ = (P₀ − P_e) / (1 − P_e)
  = (0.833 − 0.504) / (1 − 0.504)
  = 0.329 / 0.496
  = 0.663
```

**κ = 0.66 — substantial agreement** (Landis & Koch 1977 scale: 0.61–0.80)

## Disagreement Analysis

Five tasks produced conflicting labels between sessions. Each reveals a rubric ambiguity worth resolving in v0.2.

### TB-0096 — F1, hard (Day 1 FAIL → Day 2 PASS)

**What changed:** Day 1 flagged the phrase "rapidly expanding team" as a segment-specific velocity claim (HC-A violation). Day 2 cross-checked the explicit `fail_patterns` list in HC-A — "rapidly expanding" was not enumerated; only "scaling aggressively," "Series A momentum," and "speed-to-hire" were listed.

**Root ambiguity:** HC-A `fail_patterns` lists exemplary violations, not an exhaustive vocabulary. Annotators must infer whether synonyms qualify.

**Rubric fix:** Add a `synonym_classes` field grouping semantically equivalent phrases. The current list is too literal.

---

### TB-0059 — F1, medium (Day 1 PASS → Day 2 FAIL)

**What changed:** Day 1 accepted "exploring talent options" as generic exploratory language. Day 2 noticed the candidate output also contained "your continued growth trajectory" — a velocity-anchoring phrase absent from HC-A's explicit list but semantically equivalent to a disqualified pattern.

**Root ambiguity:** Same synonym gap as TB-0096, but in the opposite direction: Day 1 missed a soft violation that Day 2 caught.

**Rubric fix:** Soft dimension SD-A judge prompt should explicitly ask: "Does the email contain *any* phrase that implies the prospect's hiring velocity is known or accelerating?"

---

### TB-0122 — F2, hard (Day 1 FAIL → Day 2 PASS)

**What changed:** Day 1 treated the acqui-hire context as triggering HC-A ("recent layoff event overrides funding signal → Segment 2 or disqualified"). Day 2 re-read: HC-A applies to a *headcount reduction* layoff, not an acquisition-absorption. The target company's team was acquired intact; no personnel were cut.

**Root ambiguity:** The term "layoff event" in F2 hard constraints conflates headcount-reduction events with acquisition-absorption events. An acqui-hire preserves headcount and may indicate *more* AI investment, not less.

**Rubric fix:** Define `layoff_event.type` with values: `reduction`, `acquisition_absorption`, `restructure`. HC-A should only fire on `reduction`.

---

### TB-0132 — F3, hard (Day 1 FAIL → Day 2 PASS)

**What changed:** Day 1 flagged ACV of $450K as outside the acceptable bench band ($250K–$400K stated in bench summary). Day 2 re-read HC-B: the routing threshold is "deals over $500K require delivery-lead escalation." $450K is inside the routing threshold but above the *typical* band. The task rubric tests routing, not band compliance.

**Root ambiguity:** Two separate constraints — "typical ACV band" (a soft heuristic) and "routing threshold" (a hard rule) — appear in the same task. Day 1 applied the soft heuristic as a hard constraint.

**Rubric fix:** Separate HC entries for routing trigger vs. ACV band fit. The score formula should only penalize routing violations, not ACV-band exceptions.

---

### TB-0154 — F5, medium (Day 1 PASS → Day 2 FAIL)

**What changed:** Day 1 accepted "I'll follow up next week to check in" as a permissible scheduling statement. Day 2 flagged it: the prospect's final message contained "we're not currently looking at outside staffing options" — any follow-up scheduling violates HC-A (thread integrity: no re-approach after explicit disinterest).

**Root ambiguity:** "Not currently looking" vs. "not interested" — Day 1 interpreted "currently" as a temporal qualifier leaving the door open; Day 2 treated it as an explicit no-interest signal.

**Rubric fix:** HC-A for F5 should enumerate trigger phrases: "not currently," "not at this time," "not looking," "not interested" — all should block any follow-up commitment.

## Implications for v0.2

1. Add `synonym_classes` to F1 HC-A fail patterns.
2. Add `trigger_phrases` enumeration to F5 HC-A covering temporal hedges.
3. Separate `layoff_event.type` in F2 task schema.
4. Distinguish routing-threshold HCs from ACV-band soft heuristics in F3.

These four changes are expected to raise κ above 0.80 (almost perfect agreement) in a v0.2 re-labeling exercise.
